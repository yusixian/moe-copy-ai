# 数据收集与隐私说明

本文档说明 Moe Copy AI 扩展收集和传输的数据类型。

## 数据收集概述

**本扩展会收集网页数据并传输到 AI 服务提供商进行处理。**

根据 Firefox 的数据收集分类，本扩展使用以下数据权限：

### Required（必需）数据权限

用户必须同意以下数据收集才能使用本扩展的核心功能：

#### 1. `websiteContent` - 网页内容

**收集的数据：**
- 网页标题
- 网页正文内容
- 网页元数据（作者、发布日期等）

**用途：**
- 发送到 AI 服务进行摘要生成
- 发送到 AI 服务进行内容分析

**传输目标：**
- 用户配置的 AI API endpoint（默认 OpenAI API）

#### 2. `browsingActivity` - 浏览活动

**收集的数据：**
- 当前访问的网页 URL

**用途：**
- 作为上下文信息包含在 AI prompt 中
- 记录在本地聊天历史中

**传输目标：**
- 用户配置的 AI API endpoint（默认 OpenAI API）

## 数据传输详情

### 何时传输数据？

**仅在用户主动触发时：**
- 用户点击"生成摘要"按钮
- 用户在 AI 对话框中发送消息

**不会自动传输：**
- 浏览网页时不会自动发送数据
- 安装扩展后不会立即发送数据

### 数据发送到哪里？

**✅ 会发送到：**
- 用户配置的 AI API endpoint
  - 默认：`https://api.openai.com/v1/`
  - 可自定义为任何 OpenAI 兼容的 API

**❌ 不会发送到：**
- 扩展开发者的服务器
- 任何第三方分析或追踪服务
- 任何广告服务

### 发送的数据格式

典型的 AI API 请求包含：

```json
{
  "model": "用户选择的模型",
  "messages": [
    {
      "role": "user",
      "content": "摘要任务：提取核心观点并总结要点\n链接：https://example.com\n标题：示例文章\n内容：网页正文内容..."
    }
  ]
}
```

## 本地存储数据

以下数据**仅存储在本地浏览器**（Chrome Storage），不会传输到任何服务器：

### Chrome Sync Storage（跨设备同步）

- AI API 密钥
- AI API Base URL
- 默认系统提示词
- 模型选择
- 自定义 CSS 选择器

### Chrome Local Storage（仅本地）

- AI 聊天历史记录（最多 50 条）
  - URL
  - 用户提示词
  - AI 响应内容
  - Token 使用统计

## 用户控制

### 数据收集控制

- **必需权限**：用户必须同意数据收集才能使用 AI 功能
- **主动触发**：只有在用户点击按钮时才会发送数据

### 数据管理

- **查看历史**：用户可以在历史记录面板查看所有 AI 对话
- **删除历史**：用户可以逐条删除或清空全部历史记录
- **配置 API**：用户可以自行配置 AI API endpoint

### 退出机制

- **删除扩展**：卸载扩展会清除所有本地存储数据
- **清空历史**：在设置中可以清空聊天历史

## 第三方服务

本扩展会将数据发送到以下第三方服务：

### AI 服务提供商（用户配置）

**默认提供商：** OpenAI (https://openai.com)
- **隐私政策：** https://openai.com/privacy/
- **数据使用：** 根据 OpenAI 的隐私政策处理
- **用户可更改：** 是（支持任何 OpenAI 兼容 API）

**用户需要注意：**
- 发送到 AI API 的数据会根据该服务提供商的隐私政策处理
- 用户应自行阅读并同意其选择的 AI 服务提供商的隐私政策

## Firefox 数据收集权限声明

在 Firefox 中，安装时会显示以下数据收集权限：

```
此扩展需要收集以下数据：

必需（Required）：
- 网页内容（Website content）
- 浏览活动（Browsing activity）
```

用户必须同意这些权限才能安装和使用本扩展。
